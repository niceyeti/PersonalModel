This output of softKmeans demonstrates that this method rapidly converges
to some set of k-means, but then the soft-update methods allows "k" to diminish
monotonically. Initially, the soft update still seems intuitively correct, since it
allows fluidity between the means (as opposed to hard assignment). But once we reach
the monotonically decreasing state, the algorithm starts to fall apart. One approach might
be identifying the point when k ceases to diminish rapidly, and starts to decrease only
monotonically; in this state, it may be preferable to tune the algorithm back toward
a hard assignment of words to means, instead of allowing multiple membership.
The more mathematically sound method is probably to change the update the multiple-membership
logic, assigning word to multiple means only for certain thresholds, instead of a fixed value.
For instance, if word w belongs to mu1, mu2, mu3 with Prob 0.9,0.05,0.05, only assign w to 
mu1. But if w belongs to mu1, mu2, mu3 with Prob 0.3 0.3 0.3, then allow multiple membership;
in other words, mathematically define the "degree of membership" measure/threshold, and leverage
it to produce set consistency.

See softMeans.txt for corresponding means updates to understand the update problems
with the algorithm.
